# Screen Specifications

本書は、推定呼称探索支援アプリ「OrpheusEcho」における各画面のUI/UX仕様を定義する。
`AGENTS.md` のUI指導に基づき、各画面は以下の9つの項目で詳細化されている。

---

## 1. Home 画面

### 1.1 Purpose
- 新規の推定呼称探索セッションの開始、過去履歴の閲覧、システム設定、および教師データ用の既知名個体登録機能への入口を提供する。

### 1.2 Apple HIG Basis
- **ナビゲーション**: 標準的なフラット・ナビゲーションおよび階層型ナビゲーションの起点とする。
- **コントロール**: 大きなカード型ボタンを使用し、重要な開始アクション（新規セッション）を視覚的に最優先する。

### 1.3 NeXT / OPENSTEP / GNUstep Influence
- 画面レイアウトは厳格なグリッドシステムに基づき、クリーンな無彩色系の背景と明確な枠線でセクションを分離。構造的な「余白（Padding / Margin）」を大きく取り、視覚的な重厚感と明快さを両立させる。

### 1.4 Layout Description
- 上部：アプリタイトルおよび概要文（「推定」「有力候補」などの趣旨説明）。
- 中央：縦並びの大きなアクションボタン（「新規探索を開始」「履歴を見る」「設定」など）。
- 下部：直近のセッション履歴のサマリカード（片手操作で親指が届く位置）。

### 1.5 Interaction Behavior
- 「新規探索を開始」タップで「Species Selection（種別選択）画面」へ遷移。
- オフライン時は、過去セッションサマリカードに「未同期（Sync Pending）」インジケータをオーバーレイ表示する。

### 1.6 Visual Styling Notes
- 背景：無彩色に近い極めて薄いグレー。ボタン枠線はコントラスト比の高い細線。
- 状態カラー：同期保留中は穏やかなオレンジ、同期済みは静かなブルー（単一の色のみに依存せず、アイコン/テキストを併記）。

### 1.7 Accessibility Notes
- 全ボタンのタップ領域は縦48pt以上（HIG推奨の44pt以上を担保）。
- Dynamic Typeに対応し、システムフォントサイズ変更時にボタン内テキストが折り返して表示されるように設計。

### 1.8 Implementation Considerations
- 履歴読み込み中のスケルトン表示（Loading状態）および履歴が0件のときの文言表示（Empty状態）を実装する。

### 1.9 Self-Check Result
- 名前を「特定」する文言は一切なく、「推定」「有力候補」「参考スコア」のみを使用していることを確認した。

## 2. Species Selection (種別選択) 画面

### 2.1 Purpose
- 対象となる動物が「犬（dog）」であるか「猫（cat）」であるかを選択し、読み込む呼称辞書の初期メタデータを決定する。

### 2.2 Apple HIG Basis
- 選択肢を2つに絞り、ユーザーの認知的負荷を最小化する。標準的なセグメントコントロールではなく、より誤操作の少ない大面積の選択コントロールを採用する。

### 2.3 NeXT / OPENSTEP / GNUstep Influence
- シンプルな2カラムのボックス構造。NeXTのコントロールパネルのように、選択されたボックスに明確なインセットシャドウとアクティブベゼルを表現する。

### 2.4 Layout Description
- 画面中央を左右に2分割（または上下2分割）した巨大なカードボタン（「犬」「猫」）。
- 下部に「戻る」ボタンと「国・言語の詳細設定」へのショートカットリンクを配置。

### 2.5 Interaction Behavior
- 犬または猫のカードを選択すると即座に選択状態（枠線とハイライトが変化）となり、自動的に次の「Animal Info 画面」に遷移する。

### 2.6 Visual Styling Notes
- 犬/猫ボタンは、選択前は透過した白い背景に細い枠線、選択時はプライマリカラー（落ち着いたブルー）の枠線と背景の微細なグラデーション。

### 2.7 Accessibility Notes
- VoiceOver読み上げ時、「犬を選択」「猫を選択」と役割を明確に伝える。タップ領域は画面の約40%を占めるため、容易にアクセス可能。

### 2.8 Implementation Considerations
- SwiftUIの `HStack` または `VStack` と `Button` で構築。初期選択状態がない通常（Normal）状態と、選択後の一瞬のインタラクション（Success状態）をハンドリング。

### 2.9 Self-Check Result
- 性別やその他の不要な属性を強要しない単一の目的の画面になっている。

## 3. Animal Info (個体情報) 画面

### 3.1 Purpose
- 探索対象の動物の最低限の属性情報（保護場所、毛色、大きさ、年齢ヒントなど）およびメモを記録する。

### 3.2 Apple HIG Basis
- テキストフィールド、セレクトメニューなどの標準インプットコントロールを階層的に配置。キーボード表示時に下部ボタンが隠れないよう、スクロール追従をサポートする。

### 3.3 NeXT / OPENSTEP / GNUstep Influence
- ラベルと入力フィールドの厳密な左揃えアライメント。NeXTStepのフォーム入力フォームを模した、やや厚みのある境界線と内向きシャドウのテキストフィールド。

### 3.4 Layout Description
- すべての入力項目を1スクロール内に配置。
- 「仮ID（自動生成値）」「保護場所（テキスト）」「毛色」「年齢ヒント（若齢/成齢/高齢など）」「メモ（自由記述）」。
- 画面最下部に「探索を開始」ボタンを配置。

### 3.5 Interaction Behavior
- 各フィールドへの入力は任意（Validationルールは文字数制限のみ）。
- 「探索を開始」タップで「Exploration 画面」へ遷移。オフライン時はローカルに仮のセッションログを生成する。

### 3.6 Visual Styling Notes
- プレースホルダーテキストはコントラスト比4.5:1以上を確保したダークグレー。
- フォーカスされたテキストフィールドは、アクティブブルーの輪郭線で強調。

### 3.7 Accessibility Notes
- テキスト入力時には「キーボードの自動格納」ジェスチャー（画面タップ）をサポート。
- 各入力項目に対応する `AccessibilityLabel` を付与し、スクリーンリーダーでの入力を円滑にする。

### 3.8 Implementation Considerations
- 入力中のドラフト状態はデバイスローカル（メモリまたはキャッシュ）に保持し、不意のアプリ終了時に復旧できるようにする。

### 3.9 Self-Check Result
- 個人を特定できるような情報の入力は避け、動物の身体特徴と状況のみに限定している。

## 4. Exploration (探索実行) 画面

### 4.1 Purpose
- カメラプレビューを観察しながら、名前候補をTTS（音声合成）再生し、動物の反応（手動入力）をログとして順次記録する。

### 4.2 Apple HIG Basis
- **フィードバック**: 再生中の呼称名、現在の進捗を画面上にリアルタイムで分かりやすく表示。
- **コントロール**: 最も頻繁にタップされる「反応あり」「反応弱い」「反応なし」ボタンを、親指が最も届きやすい画面最下部に大きく配置。

### 4.3 NeXT / OPENSTEP / GNUstep Influence
- カメラプレビューの上に、半透明（Frosted Glass）のコントロールパネルが浮遊するような「階層感」を持たせたオーバーレイ設計。不要な装飾を排し、黒と暗いグレーのパネルで無機質かつ機能的な外観にする。

### 4.4 Layout Description
- 背景：全画面のカメラプレビュー（動画・写真保存の同意がある場合のみ）。
- 上部オーバーレイ：現在のセッション状況、音声再生状態、オンライン/オフラインバッジ。
- 中央オーバーレイ：現在再生中の「名前候補名」（巨大テキスト表示）および再生コントロール。
- 下部固定：手動反応ボタン（「反応あり（強）」「反応弱い」「反応なし（無反応）」）。

### 4.5 Interaction Behavior
- 「再生（スピーカー）」ボタンタップで名前を音声再生。左右スワイプで呼称候補の切り替え。
- 反応ボタンタップで、現在の試行（Trial）ログに手動反応フラグ（`manual_flag`）を保存し、自動的に次の候補名へ遷移する。
- 録画/録音は設定で明示的同意（オプトイン）を得た場合のみ実行され、プレビュー内にインジケータが表示される。

### 4.6 Visual Styling Notes
- 「反応あり」：穏やかなグリーン、「反応弱い」：イエロー、「反応なし」：ニュートラルグレー。色の見分けがつかないユーザーのため、テキスト（有・弱・無）とアイコンを必ず併記。
- 録画アクティブ時は、赤く明滅する小さな丸印インジケータを表示。

### 4.7 Accessibility Notes
- 音声ガイド（VoiceOver）作動時は、画面中央の候補名を再生と同時に音声で読み上げ、反応入力ボタンもジェスチャーで素早くタップできるように順序を最適化。
- 反応ボタンのタップ領域は60pt以上を確保。

### 4.8 Implementation Considerations
- カメラパーミッションが拒否された場合のフォールバック状態（Permission Denied状態。代わりに動物のシルエット等のプレースホルダー画像と反応記録ボタンのみを表示）を完全に実装する。
- オフライン動作時はローカルのSQLiteに試行データを即座に書き込み、キューに保存する。

### 4.9 Self-Check Result
- 「特定」や「確定」ではなく、再生した名前候補に対する「反応が強かった候補」を蓄積する設計になっている。

## 5. Candidate Ranking (候補ランキング) 画面

### 5.1 Purpose
- セッション内で試行した名前候補のうち、反応が強かった（スコアの高い）「有力候補」を順位付けして提示し、結果のレビューと再探索の準備を行う。

### 5.3 NeXT / OPENSTEP / GNUstep Influence
- リストビューはすっきりとしたボーダー線で区切られ、順位を太字の等幅フォント（Monospace）で表示。情報の「構造的な配置」を最優先したクラシカルなレポート風デザイン。

### 5.2 Apple HIG Basis
- リスト形式（`List` または `LazyVStack`）を採用。各行をタップすることで、その名前の再生時の試行詳細（反応レベルや modulation）を確認できるドリルダウン構造。

### 5.4 Layout Description
- 上部：「有力候補（参考スコア）」のタイトル。
- 中央：順位リスト。各アイテムには「候補名」「算出スコア（例: 0.91）」「反応回数」「不確実性インジケータ」。
- 下部：上位3件を基にした「愛称・近似候補の展開（再探索）」ボタン、および「結果詳細へ進む」ボタン。

### 5.5 Interaction Behavior
- 展開ボタン（`refine`）をタップすると、上位の名前候補から愛称（例：「モモ」→「モモちゃん」「もちょ」）を展開し、再探索セッションの候補リストに設定して戻る。

### 5.6 Visual Styling Notes
- スコア表示には進捗バー（Progress Bar）を併記し、視覚的に強さを把握しやすくする。
- 不確実性が高い（試行回数が極端に少ないなど）場合は、スコアの横に警告風ではない穏やかな情報アイコンを表示する。

### 5.7 Accessibility Notes
- VoiceOverが「1位 モモ、参考スコア 0.91、不確実性なし」のように構造化して読み上げるようにアクセシビリティ要素をカスタマイズ。

### 5.8 Implementation Considerations
- ランキングデータが0件（何も反応を記録しなかった場合）のEmpty状態のハンドリングを実装する。

### 5.9 Self-Check Result
- 「確定スコア」ではなく「参考スコア（スコア値）」および「有力候補」という表現を厳密に使用している。

## 6. Result Detail (結果詳細) 画面

### 6.1 Purpose
- 推定呼称の探索セッションの結果を総括し、ローカルへの完全保存、外部へのエクスポート（PDF/CSV/JSON）、および他者への共有操作を行う。

### 6.2 Apple HIG Basis
- **アクション**: システム標準の「共有シート（Share Sheet）」を呼び出し、ユーザーが使い慣れた方法（AirDrop、メッセージ、メールなど）で結果をエクスポート・共有できるようにする。

### 6.3 NeXT / OPENSTEP / GNUstep Influence
- ダイアログや共有設定オプションは、NeXTのフローティングパネル（Inspector Panel）を彷彿とさせる、フラットで境界のハッキリしたモーダルスタイル。

### 6.4 Layout Description
- 上部：セッション概要（日時、種別、記録者）。
- 中央：最終の有力候補ランキングサマリ。
- 中下部：エクスポート形式選択（PDF、CSV、JSON）と「共有する」ボタン。
- 下部：学習データ収集プログラムへのデータ提供の同意（同意する/しない）のトグルスイッチ。

### 6.5 Interaction Behavior
- 「共有する」タップ時、まずセキュリティ確認ダイアログ（「位置情報やメモ、音声データを含めますか？」）を表示し、ユーザーの選択に基づいてデータをマスクしたうえで、システム標準の共有シートを呼び出す。

### 6.6 Visual Styling Notes
- エクスポート成功時は、安心感を与える穏やかなグリーンのトーストまたはチェックマークアニメーションを一時表示する。
- データ提供の同意トグルは、ON/OFFの状態変化が明瞭なカラー（Apple標準のグリーン）を使用。

### 6.7 Accessibility Notes
- 同意トグルスイッチには、VoiceOver用の説明文「探索結果をAI改善プログラムへ提供することに同意します」を付与。

### 6.8 Implementation Considerations
- PDF生成処理はバックグラウンドで行われるため、生成中の「ローディング」インジケータと、生成完了後の「保存・ダウンロード」アクションへのステータス遷移を実装する。

### 6.9 Self-Check Result
- 個人情報、位置情報の無意識な漏洩を防ぐため、共有前にユーザーに含める項目をオプトインで選択させるステップを設けている。

## 7. History (探索履歴) 画面

### 7.1 Purpose
- 過去に実行したすべての推定呼称探索セッションの記録を一覧表示し、詳細結果の再確認や同期状態の管理を行う。

### 7.2 Apple HIG Basis
- 定番のリストビューを使用し、時系列（新しい順）で並べる。スワイプによる行削除など、iOS標準のジェスチャー操作をサポートする。

### 7.3 NeXT / OPENSTEP / GNUstep Influence
- 各履歴行は、NeXTStepのブラウザリストのように等幅で整然と並び、各要素（日付、種別、結果）が正確なタブ位置で区切られているようなビジュアル構造。

### 7.4 Layout Description
- 画面全体にスクロール可能なリストを配置。
- 各行の表示要素：
  - 左：動物アイコン（犬/猫）と実施日
  - 中央：上位3件の有力呼称候補のプレビュー
  - 右：同期ステータスアイコン（同期完了/未同期）

### 7.5 Interaction Behavior
- 履歴アイテムのタップで「Result Detail 画面」へ遷移。
- 「未同期」アイテムをタップした場合は、手動での即時同期試行オプションを表示する。

### 7.6 Visual Styling Notes
- 未同期（Sync Pending）状態は穏やかなアンバー色の丸印、同期完了は控えめなブルーのチェックマーク。

### 7.7 Accessibility Notes
- 行の削除ジェスチャーだけでなく、アクセシビリティアクションとして「このセッションを削除」ボタンを明示的に選択可能にする。

### 7.8 Implementation Considerations
- ローカルDBからの非同期読み込み。データ量が多い場合はページネーションまたは無限スクロールをサポート。
- 履歴が一切ない場合（Empty状態）は、イラストと「まだ探索履歴がありません。新しい探索を始めましょう」というガイドを表示。

### 7.9 Self-Check Result
- 履歴一覧においても「特定」などの不適切な表現がないことを確認。

## 8. Settings (設定) 画面

### 8.1 Purpose
- 音声再生（TTS）の設定、ローカル保存・同期の設定、位置情報の利用有無、および学習データ利用同意を含むプライバシー設定を統合管理する。

### 8.2 Apple HIG Basis
- グループ化されたテーブルスタイル（`Form` コントロール）を使用し、関連する設定項目（音声設定、同期設定、プライバシー）ごとにセクションを分けて直感的に整理する。

### 8.3 NeXT / OPENSTEP / GNUstep Influence
- 設定パネルはNeXTStepの「Preferences」ウィンドウの構造を模し、セクションごとに区切り線と小さなタイトルラベルを配置して、設定内容が一覧で見渡せるようにする。

### 8.4 Layout Description
- セクション1：音声設定（TTSボリューム、再生速度、デフォルト音声種別）。
- セクション2：プライバシー＆セキュリティ（カメラ使用許可、位置情報デフォルトON/OFFトグル、学習データ提供同意トグル）。
- セクション3：データ＆同期（未同期データの即時送信、ローカルキャッシュのクリア）。

### 8.5 Interaction Behavior
- トグルやセグメントコントロールの変更は、タップと同時にローカルストレージ（UserDefaults）に即時保存される。
- 「プライバシーポリシーを見る」タップでドキュメントビューをポップアップ表示。

### 8.6 Visual Styling Notes
- トグルスイッチなどのシステムコントロールはAppleのシステムカラーを使用し、一貫性を保つ。

### 8.7 Accessibility Notes
- 各設定項目は十分なコントラスト比を維持し、VoiceOverではトグルの現在の状態（オン/オフ）と設定の役割を分かりやすく説明する。

### 8.8 Implementation Considerations
- プライバシー関連のトグルを変更した際、変更内容を反映するために必要であればシステム権限設定へのダイレクトリンクを表示する。

### 8.9 Self-Check Result
- 位置情報や録画等の危険な初期値を避け、プライバシー・ファーストの設定項目がデフォルトとなっている。
