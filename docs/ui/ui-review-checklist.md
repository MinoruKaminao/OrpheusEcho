# UI Review Checklist

## Purpose

- [ ] 画面の主目的が1つに絞られている
- [ ] 主操作が明確である
- [ ] 副操作が主操作を邪魔していない

## Apple HIG

- [ ] 標準コンポーネントを優先している
- [ ] 標準ナビゲーションに従っている
- [ ] iPhoneで片手操作しやすい
- [ ] iPadで情報密度を適切に調整している

## State Design

- [ ] 通常状態
- [ ] 読み込み中
- [ ] 空状態
- [ ] エラー
- [ ] 権限不足
- [ ] オフライン
- [ ] 同期待ち
- [ ] 成功

## Accessibility

- [ ] Dynamic Type対応
- [ ] VoiceOverラベル
- [ ] 44pt以上のタップ領域
- [ ] 十分なコントラスト
- [ ] 色だけで意味を示していない

## Domain Safety

- [ ] 「確定」「正解」「特定」を避けている
- [ ] 「推定」「有力候補」「参考」を使用している
- [ ] 撮影・録音・位置情報の同意が明確である
